Funções principais de um SGBD?

Definição de Dados (DDL - Data Definition Language): Permite definir a estrutura do banco de dados, como criar, alterar e excluir tabelas, índices, visões, etc. Manipulação de Dados (DML - Data Manipulation Language): Facilita a inserção, atualização, exclusão e consulta dos dados armazenados. Controle de Acesso e Segurança: Gerencia permissões de usuários para garantir que apenas pessoas autorizadas possam acessar ou modificar os dados. Integridade dos Dados: Assegura que os dados sejam consistentes e válidos através de restrições e regras, como chaves primárias e estrangeiras. Controle de Concorrência: Permite que múltiplos usuários acessem e modifiquem os dados simultaneamente sem causar inconsistências. Recuperação e Backup: Garante a restauração dos dados em caso de falhas, erros ou desastres, mantendo a integridade do banco. Armazenamento e Organização: Gerencia como os dados são fisicamente armazenados, organizados e acessados para otimizar performance. Suporte a Linguagem de Consulta (SQL): Permite a execução de comandos e consultas para manipulação e recuperação dos dados.
