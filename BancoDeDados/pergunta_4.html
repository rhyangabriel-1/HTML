Tipos(Relacional e não relacional)

SGBD Relacional Organiza os dados em tabelas (linhas e colunas). Usa chaves para relacionar informações entre tabelas. Linguagem usada: SQL (Structured Query Language). Exemplos: MySQL, PostgreSQL, SQL Server, Oracle. Exemplo: Tabelas de Clientes e Pedidos, onde cada pedido está ligado a um cliente pelo ID. 2. SGBD Não Relacional (NoSQL) Armazena dados de forma flexível, sem usar tabelas. Ideal para grandes volumes de dados ou informações sem estrutura fixa (como redes sociais, sensores, etc.). Tipos: documentos, chaves-valor, grafos, colunas. Exemplos: MongoDB, Cassandra, Redis, Neo4j.

omparação entre SGBD Relacional e Não Relacional Característica Relacional Não Relacional Estrutura Tabelas (linhas e colunas) Documentos, grafos, pares chave-valor Linguagem SQL Diversas (ex: JSON, APIs) Escalabilidade Vertical (aumenta poder do servidor) Horizontal (aumenta número de servidores) Flexibilidade Estrutura fixa Estrutura flexível Indicado para Dados estruturados e consistentes Dados grandes e variados Exemplos MySQL, PostgreSQL MongoDB, Cassandra
